objects
Each physical thing in the world can be represented as an object. Each object can have its own properties, events, and methods.
example: var person = { firstName:"Laura", lastName:"DeGroot", age:25 };
properties
Characteristics of an object. Each property has a name and a value, and each of these name/value pairs tells you something about each individual instance of the object.
example: var person = { firstName:"Laura", lastName:"DeGroot", age:25 };
events
Programs are designed to do different things when users interact with the computer in different ways. An event is the computer's way of sticking up its hand and saying "Hey, this just happened!"
methods
Methods perform tasks using the properties of an object. They typically represent how people or other things interact with an object in the real world.
example: checkName()
document object
The document object represents an HTML page. Like other objects, the document object has properties, methods and events. Properties describe characteristics of the current web page (such as the title). Methods perform tasks associated with the document currently loaded in the browser (such as getting new information from an element or adding content). Events can be responded to, such as a user clicking on an element.
interpreted programming language
An interpreter (scripting engine) takes your instructions (eg. in JavaScript) and translates them into instructions the browser can use. In an interpreted programming laguage, each line of code is translated one by one.
member
Methods and properties of an object are called members.
member operator
You can access the members of an object using a dot between the object name and the member you want to access. It's called a member operator.
example: document.write('hello');
parameter
Whenever a method requires some information in order to work, the data is given inside the parentheses. Each piece of information is called a parameter.
example: document.write('hello');
statement
A statement is an individual instruction that the computer should follow. Each one should start on a new line and end with a semi colon.
example: if (hourNow > 18){ greeting = "Good evening"; } else { greeting = "Welcome"; }
multi-line comments
To write a comment that stretches over more than one line, you can use a multiline comment, starting with the /* characters and ending with the */ characters
example: /* hey check me out I'm a
muli-line comment/*
single-line comments
In a single-line comment, anything that follows the two forward slash characters // on that line will not me processed by the JavaScript interpreter.
example: // hey check me out I'm a single line comment
variable
A script will have to temporarily store the bits of information it needs to do its job. It can store this data in variables. If a variable is more than one word, it is usually written in camelCase.
example: var myAge = 25;
data types
Numeric data type: numbers. String data type: letters and other characters. Boolean data type: true or false.
